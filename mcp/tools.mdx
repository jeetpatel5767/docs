## audit_move_source

Audit raw Move source code pasted directly into the conversation. No on-chain interaction required.

### Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `source` | `string` | ✅ | Move source code (one or more modules) |
| `filename` | `string` | No | Filename hint — default: `contract.move` |
| `network` | `"testnet" \| "mainnet"` | No | Sui network context — default: `testnet` |

### Example prompt

> *"Audit this Move contract and tell me the critical findings:"*
> ```move
> module vault::vault {
>   public entry fun withdraw(pool: &mut Pool, amount: u64, ctx: &mut TxContext) {
>     // no capability check
>   }
> }
> ```

## audit_package_id

Fetch and audit a live on-chain Sui package by its address. Pulls all modules via Sui GraphQL — no source upload needed.

### Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `packageId` | `string` | ✅ | Sui package address — `0x` + 64 hex chars |
| `network` | `"testnet" \| "mainnet"` | No | Sui network — default: `testnet` |

### Example prompt

> *"Audit the Cetus CLMM package on mainnet: `0xa9b0ffe2f8e713a66ad1aa361cf1984526a5048c6de786b4dd292f3eed204b92`"*

This audits the exact package that suffered the $223M exploit. MoveLens detects `ML-INT-001` (the Cetus-class bit-shift overflow) on Layer 1 in under 5 seconds.

## audit_github_repo

Clone a public GitHub repository and audit all `.move` files (up to 50).

### Parameters

| Parameter | Type | Required | Description |
|---|---|---|---|
| `repoUrl` | `string` | ✅ | Public GitHub repo URL |

### Example prompt

> *"Audit this Move protocol before I integrate with it: `https://github.com/owner/sui-defi`"*

### Notes

- Only public repos are supported
- Directories `build/`, `target/`, `.git/`, `deps/`, `node_modules/` are excluded
- Repos with more than 50 `.move` files are capped at 50 (the `cappedAt` field is set in the report)

## Report format

Every tool returns a markdown report structured like this:

```markdown
## MoveLens Security Report

**Package:** @cetus/amm
**Network:** mainnet
**Risk Grade:** 🔴 **F**
**Findings:** 34 total (1 critical · 12 high · 18 medium · 3 low)
**Layers:** Layer 1 (rules) + Layer 2 (OZ) + Layer 4 (ML) + Layer 3 (LanceDB)
**Walrus blob:** `5cN1fBWk5TIXUlJv-Do7pWYjc3AknRCJ-buNt_JdnPA`
**Full report:** http://16.171.224.235:3000/audit/7f3a1c2d-...

> Automated pre-screen — not a substitute for a human audit.

---

### 1. 🔴 [CRITICAL] ML-INT-001
**Module:** `integer_mate` · **Line:** 42 · **Confidence:** 100%
**Issue:** Bitwise shift with wrong overflow mask — Cetus class ($223M loss May 2025).
**Fix:** Replace `0xffffffffffffffff << N` mask with `1u256 << N`.

```move
// ✅ Suggested fix:
let result = (1u256 << shift) - 1;
```
```

The first 20 findings are shown inline. A link to the full browser report is included when there are more.
