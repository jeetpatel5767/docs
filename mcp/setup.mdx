## What you get

Once connected, any Claude conversation can call three native tools:

| Tool | What it does |
|---|---|
| `audit_move_source` | Paste raw Move code → instant security report |
| `audit_package_id` | On-chain package address → full audit via Sui GraphQL |
| `audit_github_repo` | Public GitHub URL → audits every `.move` file in the repo |

---

## Claude Desktop — 2 steps

<Steps>
  <Step title="Add MoveLens to your config file">
    Open your Claude Desktop config:
    - **macOS:** `~/Library/Application Support/Claude/claude_desktop_config.json`
    - **Windows:** `%APPDATA%\Claude\claude_desktop_config.json`

    Paste this (create the file if it doesn't exist):

    ```json
    {
      "mcpServers": {
        "movelens": {
          "url": "http://16.171.224.235:3000/api/mcp"
        }
      }
    }
    ```

    That's it — no local server, no npm install, no cloning required.
  </Step>

  <Step title="Restart Claude Desktop and try it">
    Fully quit and relaunch Claude Desktop. The MoveLens tools appear automatically in the tool picker (hammer icon).

    Try it instantly:

    > *"Audit this Move contract:"* → paste any `.move` source

    or

    > *"Audit the Cetus CLMM package: `0xa9b0ffe2f8e713a66ad1aa361cf1984526a5048c6de786b4dd292f3eed204b92` on mainnet"*

    Claude calls `audit_package_id` and returns a full security report inline — risk grade, all findings with line numbers and suggested fixes, and a permanent Walrus blob ID.
  </Step>
</Steps>

## Claude Code — 1 command

```bash
claude mcp add movelens --url http://16.171.224.235:3000/api/mcp
```

The tools are then available in every `claude` session within that project.

---

## How it works

```
Claude Desktop / Claude Code
        │  MCP tool call (HTTP)
        ▼
http://16.171.224.235:3000/api/mcp
        │  submits audit job
        ▼
  MoveLens audit engine
  ├─ Layer 1: 65 regex rules
  ├─ Layer 2: 10 OZ benchmarks
  ├─ Layer 3: LanceDB semantic recall
  └─ Layer 4: Groq llama-3.3-70b (optional)
        │  stores report
        ▼
     Walrus blob ← permanent on-chain
        │  returns markdown report
        ▼
     Claude response (inline)
```

The MCP server is a thin HTTP bridge hosted alongside the audit engine. No state is stored in the MCP layer — all analysis and storage happens in the backend.

---

## Timeouts

Audits complete in **10–60 seconds** for most contracts. The MCP server polls every 3 seconds and times out after 3 minutes. If an audit times out, retry — it's idempotent.

---

<Note>
  Layer 4 (Groq ML classification) requires a `GROQ_API_KEY` to be set on the server. Without it, the pipeline runs on Layers 1–3 and remains fully functional. All findings from Layers 1 and 2 are deterministic and confidence 1.0.
</Note>
