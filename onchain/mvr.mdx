## What is MVR?

The [Move Version Registry](https://docs.sui.io/concepts/sui-move-concepts/packages/version-registry) is a Sui on-chain registry that maps human-readable package names (e.g. `@deepbook/core`) to package addresses and versioned metadata.

Each registered package has a `PackageInfo` object. MoveLens calls `set_metadata` on that object to store the Walrus blob ID as a key-value entry — creating a permanent, verifiable link: **"this package was audited; here is the proof."**

---

## How it works

```typescript
// src/lib/mvr/metadata.ts
attachAuditToPackage({
  packageInfoId: "0xcc7af44f...",  // PackageInfo object ID
  blobId:        "5cN1fBWk5T...", // Walrus blob ID
  signer:        keypair,
})
→ { txDigest: "85rKhcZx..." }
```

The function:
1. Reads the current `PackageInfo` metadata via GraphQL to check if `"movelens_audit"` key already exists
2. If it exists: calls `unset_metadata` first (vec_map aborts on duplicate key insert)
3. Calls `set_metadata("movelens_audit", blobId)`
4. Waits for indexer confirmation with retry (up to 6 attempts, 3s delay — indexer lag is common)

---

## Opt-in only

MVR attestation requires the user to **check "Publish audit on-chain"** in the audit form. This sets `publishOnChain: true` in the request body.

If `publishOnChain: false` (default):
- Stage 5 (MVR attach) is skipped
- `txDigest` is `null` in the report
- `mvr_name` is excluded from the public Walrus quilt metadata
- The trust panel shows "Not published on-chain (opt-in)"

## MVR name resolution

`src/lib/mvr/resolve.ts` provides `resolvePackageName(packageId)` — a reverse lookup from package address to MVR name (e.g. `0x2eeaab...` → `@deepbook/core`). This populates `mvrName` in the report header. Unregistered packages return `null`; the address is shown instead.
