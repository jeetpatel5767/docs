# MoveLens

Security auditing for Sui Move smart contracts. Deterministic rule engine, OpenZeppelin benchmarks, LanceDB semantic recall, and a Groq LLM ensemble — unified into one pipeline that stores permanent encrypted reports on **Walrus**. Built for Sui Overflow 2026 (Walrus track).

## What you get

- **Layer 1** — 65 compiled regex rules + 19 AST stubs across 13 Move vulnerability sectors
- **Layer 2** — 10 OpenZeppelin-class pattern-absence checks (what's missing, not just what's wrong)
- **Layer 3** — LanceDB vector memory of 52 historically exploited Move snippets, grows as you audit
- **Layer 4** — Groq `llama-3.3-70b-versatile` classifies ambiguous snippets; Jina embeddings boost confidence on corpus matches
- **Walrus storage** — permanent blob per audit; quilt contains `report.json`, `findings.enc`, `summary.md`
- **Seal encryption** — IBE threshold encryption on the findings entry; only the package owner can decrypt
- **MVR attestation** — optional `set_metadata` call links the blob to a `PackageInfo` object on-chain
- **GitHub repo auditing** — paste a URL, MoveLens clones it, finds all `.move` files, runs the full pipeline
- **MCP server** — three tools for Claude Desktop and Claude Code: source, package ID, GitHub repo
- **SQLite job store** — audit jobs survive server restarts; auto-pruned after 24 hours

## How the pieces fit

```
Input (address / source / GitHub URL)
  │
  ├─ Layer 1: 65 regex rules → findings (confidence 1.0)
  ├─ Layer 2: 10 OZ checks  → findings (confidence 0.95)
  ├─ Layer 3: LanceDB recall → memory context injected into Layer 4 prompt
  └─ Layer 4: Groq + Jina   → findings (confidence model-dependent, severity-floored)
       │
       └─ Engine: merge + dedupe + severity floor + sort (Cetus-class first)
            │
            └─ AuditReport → Seal encrypt → Walrus quilt → blobId
                                                │
                                                └─ (optional) MVR set_metadata → txDigest
```

The Cetus CLMM package (`0xa9b0ffe2...`, mainnet) fires rule `ML-INT-001` **34 times** — one bit-shift error that caused a $223M loss in 2024. MoveLens catches it in under 5 seconds on Layers 1 + 2 alone.
