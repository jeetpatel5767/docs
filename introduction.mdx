## Overview

MoveLens is an open-source security auditor for [Sui Move](https://docs.sui.io/guides/developer/first-app/write-package) smart contracts. It runs a **four-layer hybrid analysis pipeline** on any Move package — from a live on-chain address, pasted source code, or a public GitHub repository — and produces a permanent, encrypted audit report stored on **Walrus decentralised storage**.

Built for the **Sui Overflow 2026 hackathon** (Walrus track), MoveLens is the first auditor to integrate:

- Deterministic regex + AST rule engine (Layer 1)
- OpenZeppelin benchmark comparisons (Layer 2)
- LanceDB semantic memory recall (Layer 3)
- Groq `llama-3.3-70b-versatile` + Jina embedding ensemble (Layer 4)
- **Walrus** blob storage for permanent, verifiable reports
- **Seal** threshold encryption so only the package owner can decrypt findings
- **MVR** on-chain attestation linking a Walrus blob to a Sui package

## Why MoveLens?

### The problem

Sui Move is memory-safe and object-centric, but move-specific vulnerabilities — integer overflow via unchecked bitwise shifts, capability object misuse, hot-potato flash loan abuse — have caused **hundreds of millions in losses** (Cetus: \$223M, 2024).

Existing Solidity auditors don't understand Move's object model. Human audits are expensive and slow. There was no open, automated pre-screening tool for Move.

### The solution

MoveLens sits between "no audit" and "full human engagement." It:

1. **Screens instantly** — 65 deterministic regex rules catch the most common patterns in milliseconds.
2. **Benchmarks against OZ** — 10 OpenZeppelin-class deviation checks verify safe patterns are present.
3. **Recalls past vulnerabilities** — 52-snippet LanceDB corpus finds code that resembles historically exploited patterns.
4. **Uses a real LLM for context** — Groq `llama-3.3-70b-versatile` (free tier) classifies ambiguous snippets with natural-language reasoning.
5. **Stores reports on-chain** — every report is a Walrus blob with a permanent ID, optionally Seal-encrypted and MVR-attested.

<Warning>
  MoveLens is an automated pre-screen. It **is not a substitute for a human audit**. This watermark appears verbatim on every report and cannot be removed.
</Warning>

## The Cetus exploit — why this matters

In 2024 the Cetus protocol lost **\$223M** to a single integer overflow: `checked_shlw` (checked shift-left-wrapping) was used instead of the safe `checked_shl`. The difference is one character. Every deposit, withdrawal, and swap function was affected across 34 call sites.

MoveLens catches this. Run the Cetus CLMM package (`0xa9b0ffe2...`) through the auditor: rule `ML-INT-001` fires at every instance, risk grade returns **F**, and the report links to the Walrus blob that proves the audit happened.
